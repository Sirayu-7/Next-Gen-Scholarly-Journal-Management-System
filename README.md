#1. Project Name
-"ModernJO: Next-Gen Scholarly Journal Management System"

##2. Features (อ้างอิงจากขั้นตอนในเอกสาร )
 -ระบบจะแบ่งฟีเจอร์ตามสถานะของบทความ (Workflow Stages) ดังนี้:
  -Submission Management (ระบบจัดการบทความส่งเข้า):
  -หน้า Dashboard สำหรับบรรณาธิการเพื่อดูบทความที่ส่งเข้ามาใหม่ (Unassigned).

ระบบมอบหมายผู้ดูแลบทความ (Add Participants) ให้กับกองบรรณาธิการ.

Peer Review Workflow (กระบวนการประเมินโดยผู้ทรงคุณวุฒิ):

ระบบจัดการ Reviewer: ค้นหาและมอบหมายผู้ประเมิน พร้อมกำหนดวันตอบรับ (Response Due Date) และวันส่งผล (Review Due Date).
+1

ระบบ Blind Review: การอัปโหลดไฟล์ที่ตัดชื่อผู้แต่งออกเพื่อความเป็นธรรม.
+1

Decision Making: บรรณาธิการสามารถตัดสินใจได้ 4 ทางหลัก คือ รับตีพิมพ์ (Accept), ให้แก้ไข (Revisions Required), ส่งประเมินใหม่ (Resubmit), หรือปฏิเสธ (Decline).
+1

Copyediting & Production (การปรับปรุงและเตรียมตีพิมพ์):

ระบบตรวจสอบความถูกต้อง (Proofreading) และจัดหน้า (Layout).

ระบบ Galley Management: จัดการไฟล์ PDF สำหรับการเผยแพร่ออนไลน์.
+2

Metadata Editing: แก้ไขบทคัดย่อและคำสำคัญให้ถูกต้องก่อนเผยแพร่.
+1

Communication Hub:

ระบบ Discussion ภายในสำหรับโต้ตอบระหว่าง Author, Editor และ Reviewer ในแต่ละขั้นตอน.
+1

Email Notification: ระบบส่งอีเมลแจ้งเตือนอัตโนมัติเมื่อมีการเปลี่ยนแปลงสถานะ.
+2

3. Color Palette & UI Concept
ใช้โทนสีตามที่คุณกำหนดเพื่อสร้างความน่าเชื่อถือและความทันสมัย:

Primary (#0E21A0 - 60%): ใช้สำหรับ Navigation Bar, Sidebar และส่วนหัวของเว็บไซต์เพื่อให้ดูมีความเป็นวิชาการและมั่นคง

Secondary (#4D2FB2 - 20%): ใช้สำหรับปุ่มดำเนินการหลัก (Primary Actions) เช่น "Send to Review" หรือ "Accept Submission"

Accent (#B153D7 - 10%): ใช้สำหรับสถานะที่กำลังดำเนินการ (In-progress) หรือส่วนที่ต้องการเน้นความสนใจ

Highlight (#F375C2 - 10%): ใช้สำหรับ Notification หรือการแจ้งเตือนสำคัญที่ต้องการให้เห็นเด่นชัด
